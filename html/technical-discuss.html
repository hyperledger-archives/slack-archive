benedikt.herudek
2016-04-09 04:05
Hi all, find below a presentation I held this week at the San Diego Linux IoT summit on using Blockchains for connecting disparate protocols in a transaction system. It introduces a concept called 'Proof of Understanding', intended to (partially) replace proof of work for creating immutability and allowing different machines and applications with different languages and protocols to interact. Basic technical idea is: we use a blockchain as a supervised training set to predict future message formats based on previous transactions message formats. For new Connections sender & receiver have to run through an agreement and negotiation process (format, content, action handshake) to agree on how they communicate. The Miner (called here Translator) is responsible for suggesting message formats and upon success gets rewarded in a cryptocurrency. Successful and un-successful handshakes together with relevant features allowing predictions based on Machine Learning algorithms get recorded in the Blockchain. Partially replacing 'proof of work' with 'proof of understanding' (calling the resulting System Babelchain after Hitchhiker's babelfish) will allow a smarter and maybe lesser use of Machine energy consumption and via connecting before hands unconnected systems has the potential to disrupt some quasi monopolistic information silos. This idea is in a white paper status and would need to be prototyped. I'd be interested in hearing people's PoV on the suggestion. whitepaper (consider this leading) https://github.com/Benudek/babelchain/blob/master/Proof%20of%20Understanding%20%E2%80%93%20Babelchain%20Machine%20Communication%20.pdf presentation held at Linux IoT summit partially outdated, whitepaper leading) https://github.com/Benudek/babelchain/blob/master/IoT%20Babelchain%20-%20A%20Blockchain%20Solution%20to%20the%20'Baskets%20of%20Remotes'%20Problem%20FINAL.pdf regards

@dmurik - In short, complete history of the state is not maintained.
